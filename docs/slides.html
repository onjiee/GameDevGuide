GitHub Repo

📁 โฟลเดอร์โปรเจกต์

เก็บไฟล์ทั้งหมดไว้บน GitHub เหมือนโฟลเดอร์บนเครื่อง แต่อยู่บน cloud

GitHub Pages

🌐 เปิดเป็นเว็บได้เลย

GitHub เอาไฟล์ใน repo ไปแสดงเป็นเว็บไซต์ให้ทุกคนเข้าได้ ฟรี!

URL ที่ได้: username.github.io/ชื่อ-repo/
⚙️ ตั้งค่าที่ไหน? → Settings → Pages → Branch: main → เลือก Folder
Folder ที่ให้เลือกมี 2 แบบ: / (root) และ /docs — สไลด์นี้จะสอนทั้งสองแบบ

🚪 index.html = ประตูหน้า

เปรียบเทียบหน้า 5 / 10

Root vs Docs — เลือกแบบไหนดี?

📂 Root Mode ( / )

✓
เริ่มต้นง่ายที่สุด
วางไฟล์ที่ root ได้เลย
✓
เหมาะกับ repo ที่ทำเว็บอย่างเดียว
ไม่มีโค้ดอื่นปะปน
✗
ไฟล์รก ถ้ามีโค้ดอื่นด้วย
Python, README, .py ปะปนกับ HTML

📁 Docs Mode ( /docs )

✓
แยกไฟล์เว็บออกจากโค้ดอื่น
docs/ = เว็บ / ที่เหลือ = โค้ด
✓
เหมาะกับ repo ที่มีทั้งเว็บ + backend
เช่น มี Python, Node.js ด้วย
✗
ต้องจำว่าทุกอย่างอยู่ใน docs/
ลืมใส่ใน docs/ → เว็บไม่อัปเดต
💡 ถ้าทำแค่เว็บอย่างเดียว → ใช้ Root ก็พอ / ถ้ามีโปรเจกต์อื่นด้วย → ใช้ Docs
กฎสำคัญหน้า 6 / 10

2 วิธี Link ไป Labหน้า 8 / 10

เชื่อมจาก Homepage ไป Lab — 2 วิธี

วิธีที่ 1 — Redirect (กดปุ๊บไปเลย)

index.html ใน Lab01/ มีแค่บรรทัดเดียว กดลิงก์แล้วไปเกมทันที

<!-- Lab01/index.html --> <meta http-equiv="refresh"       content="0; url=lab01.html">
✓ ง่าย ไม่ต้องทำหน้าพิเศษ
คนใช้กดลิงก์ → เกมโหลดเลย

วิธีที่ 2 — Intro Page (มีหน้าแนะนำ)

index.html ใน Lab01/ มีหน้าอธิบาย Lab พร้อมปุ่มเล่นและปุ่มกลับ

<!-- Lab01/index.html --> <h1>Lab 01 — ชื่องาน</h1> <p>คำอธิบายสั้นๆ</p> <a href="lab01.html">▶ เล่น</a> <a href="../">← กลับ</a>
✓ ดูเป็นมืออาชีพกว่า
../ = ขึ้นไป 1 folder (กลับ root)
โค้ด Homepageหน้า 9 / 10

Homepage — โค้ดเชื่อมไป Lab แต่ละอัน

index.html ที่ root (หรือ docs/) — รายการลิงก์ไปแต่ละ Lab
<ul>   <li><a href="Lab01/">Lab 01</a></li> ← แบบ folder   <li><a href="Lab02/">Lab 02</a></li>   <li><a href="Lab03.html">Lab 03</a></li> ← แบบ .html ตรงๆ </ul>
⚠️ โค้ดที่ผิด — tag ปิดไม่ครบ
<!-- ผิด: <a> อยู่นอก <li> และ <li></li> ว่างเปล่า --> <ul>   <li></li><a href="lab01.html">Lab01</a> </ul>
✏️ แก้ให้ถูก: <li> ต้องครอบ <a> ไว้ข้างใน และต้องปิด </li> ทุกอัน
